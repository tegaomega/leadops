Overview

Executive Summary

LeadOps is a full-stack, real-time lead intelligence and outreach CRM engineered for a Wix Studio web development business targeting Houston's undigitized SMB market. The system operationalizes a complete data pipeline: automated discovery of no-website businesses via Google Maps Places API → proprietary lead scoring → real-time team collaboration → structured outreach execution.

The platform eliminates manual workflows at every stage — from lead acquisition to team coordination — replacing spreadsheets and fragmented tools with a unified, live system shared across a 3-person sales team.

Data Integrity

Idempotency & Duplicate Safety

Google assigns every business a permanent place_id. The schema enforces UNIQUE NOT NULL on this column. Every write uses PostgreSQL upsert with ON CONFLICT DO UPDATE.

-- Schema constraint place_id TEXT UNIQUE NOT NULL -- Python upsert (runs idempotently) "Prefer": "resolution=merge-duplicates" # Result: re-running the script 10 times = same data # Stale fields (phone, rating, reviews) are refreshed # Zero duplicate rows ever created
Real-Time

Live Collaboration Architecture

Supabase implements PostgreSQL logical replication via WebSocket pub/sub. The useLeads hook subscribes to INSERT and UPDATE events, merging CDC payloads into React state with optimistic updates.

// Real-time subscription supabase.channel('leads-realtime') .on('postgres_changes', { event: 'INSERT' }, (payload) => { setLeads(prev => [payload.new, ...prev]) }) .on('postgres_changes', { event: 'UPDATE' }, (payload) => { setLeads(prev => prev.map(l => l.id === payload.new.id ? payload.new : l)) }) .subscribe()

All 3 team members see kanban moves, note updates, and new leads within ~200ms — no polling, no refresh required.
